# CONTRIBUTING.md

Цель: Руководство для разработчиков и аналитиков, которые хотят добавить новый плагин, навык или инструмент в маркетплейс PM IQ.
План содержимого:

## 1. Getting Started
- [ ] Как сделать форк репозитория и настроить локальное окружение.
- [ ] Установка зависимостей для валидации: `pip install pyyaml jsonschema`.

## 2. How to Create a New Plugin
- [ ] Шаг 1: Создать папку `plugins/my-new-plugin/`.
- [ ] Шаг 2: Создать `README.md` с описанием.
- [ ] Шаг 3: Создать `.mcp.json` с объявлением инструментов и переменных окружения.
- [ ] Шаг 4: Создать папку `skills/my-skill/` и файл `SKILL.md` с YAML frontmatter.
- [ ] Шаг 5: Создать файл мок-данных `skills/my-skill/my_tool_name.csv`.

## 3. Data & Mocking Standards
- [ ] Правила именования CSV-файлов (должны совпадать с именем инструмента).
- [ ] Требования к заголовкам CSV (использовать `snake_case`, избегать пробелов).
- [ ] Как добавить новый столбец в CSV и убедиться, что `unified_mock_server.py` его корректно фильтрует.

## 4. Validation & Testing
- [ ] Обязательный шаг: запуск `python scripts/validate_marketplace.py`.
- [ ] Как проверить плагин локально с помощью `samples/pm_iq_agent.py`.

## 5. Pull Request Guidelines
- [ ] Что должно быть в описании PR (цель изменения, затронутые плагины, результаты валидации).
- [ ] Требования к ревью кода (проверка длины description, корректность JSON).
