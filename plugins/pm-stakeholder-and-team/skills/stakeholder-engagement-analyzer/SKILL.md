# Stakeholder Engagement Analyzer

Этот скилл поддерживает домены производительности "Команда" (Team) и "Заинтересованные стороны" (Stakeholders).

## Когда использовать
- При вопросах о том, доволен ли заказчик, почему команда задерживает задачи или как улучшить коммуникацию.

## Инструкции для агента
1. Запроси реестр стейкхолдеров через `get_stakeholder_register` и оцени их текущий уровень вовлеченности vs. желаемый.
2. Используй `analyze_communication_patterns` (на основе метаданных почты/мессенджеров), чтобы выявить "информационные вакуумы" или, наоборот, информационный шум.
3. Для оценки команды используй `get_feedback_summary` и метрики загрузки. Если видишь признаки выгорания (чрезмерные сверхурочные, рост количества дефектов), немедленно флажь это как риск.
4. Предлагай конкретные действия: "Заказчик не открывал последние 3 отчета. Рекомендую назначить короткую встречу для синхронизации ожиданий".

## Используемые MCP инструменты
- `get_stakeholder_register`
- `analyze_communication_patterns`
- `get_meeting_decisions`
- `get_feedback_summary`
