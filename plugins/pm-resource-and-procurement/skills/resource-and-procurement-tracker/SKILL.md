# Resource and Procurement Tracker

Интегрированное управление всеми типами ресурсов.

## Методологический контекст

### Матрица критичности блокёров

| Тип ресурса | Критерий | Приоритет |
|---|---|---|
| Люди | Перегрузка > 100% + на критическом пути + уникальная компетенция | 🔴 КРИТИЧЕСКИЙ |
| Люди | Перегрузка > 100% + на критическом пути | 🟠 ВЫСОКИЙ |
| Люди | Перегрузка > 100% + длительность > 3 недели | 🟠 ВЫСОКИЙ |
| Оборудование | Доступность < 70% + простой > 10 дней | 🔴 КРИТИЧЕСКИЙ |
| Оборудование | Доступность < 85% | 🟡 СРЕДНИЙ |
| Материалы | Задержка > 5 дней + на критическом пути | 🔴 КРИТИЧЕСКИЙ |
| Материалы | Задержка > 5 дней | 🟠 ВЫСОКИЙ |
| Подрядчики | KPI < 75% + задержки > 20% | 🔴 КРИТИЧЕСКИЙ |
| Подрядчики | KPI < 85% | 🟡 СРЕДНИЙ |

## Инструкции для агента

### Шаг 1 — Получи комплексную картину
Вызови все инструменты для полного анализа:
- `get_resource_histogram` → загрузка людей, критический путь, уникальные компетенции
- `get_equipment_availability` → доступность оборудования, простои, загрузка
- `get_material_delivery_schedule` → статус материалов, задержки (variance_days)
- `get_contractor_performance` → производительность подрядчиков, надёжность поставок

### Шаг 2 — Выяви блокёры

**Люди (get_resource_histogram):**
- Перегрузка > 100% + `on_critical_path = true` → критический риск срыва сроков
- Перегрузка > 100% + `overload_duration_weeks > 3` → риск выгорания
- `unique_critical_skill = true` + перегрузка → риск "единственной точки отказа"

**Оборудование (get_equipment_availability):**
- `availability_pct < 70%` + `downtime_days > 10` → критический простой
- `availability_pct < 85%` → требует внимания
- `utilization_pct < 50%` → недогрузка, возможность перераспределения

**Материалы (get_material_delivery_schedule):**
- `variance_days < -5` → задержка поставки
- `status = Delayed` → проверить влияние на критический путь

**Подрядчики (get_contractor_performance):**
- `kpi_score_pct < 75%` → критическое падение эффективности
- `delayed_deliveries / total_deliveries > 0.2` → ненадёжный поставщик
- `avg_delay_days > 7` → системные задержки

### Шаг 3 — Оцени влияние
- Блокёр на критическом пути → немедленная эскалация
- Множественные блокёры → пересмотр плана
- Точка отказа + перегрузка → план преемственности + снижение нагрузки

### Шаг 4 — Предложи действия

**Люди:**
- Перегрузка на критическом пути → resource leveling, crashing, найм
- Уникальная компетенция + перегрузка → план преемственности, аутсорсинг как страховка
- Длительная перегрузка → перераспределение задач, отпуск

**Оборудование:**
- Критический простой → аренда замены, ускоренный ремонт
- Низкая загрузка → перераспределение на другие задачи
- Плановое ТО → синхронизация с расписанием проекта

**Материалы:**
- Задержка > 5 дней → поиск альтернативного поставщика, ускоренная доставка
- Критическая задержка на критическом пути → эскалация, штрафные санкции

**Подрядчики:**
- KPI < 75% → штрафные санкции, замена подрядчика
- Высокий процент задержек → пересмотр контракта, поиск альтернативы
- Низкое качество → усиление контроля, требования к исправлению

### Шаг 5 — Выбери виджет

**Люди:**
- "Кто перегружен?" → `resource_readiness`
- "Кто на критическом пути?" → `critical_path_resources`

**Оборудование:**
- "В каком состоянии оборудование?" → `equipment_health`

**Материалы:**
- "Какие материалы задерживаются?" → `material_delay_breakdown`

**Подрядчики:**
- "Какие подрядчики ненадёжны?" → `contractor_delivery_reliability`

**Действия:**
- "Нужен план реагирования?" → `resource_action` (один виджет на одну проблему)
