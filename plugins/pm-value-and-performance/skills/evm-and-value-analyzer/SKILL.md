# EVM and Value Analyzer

Реализует домен производительности **Measurement** и принцип **Value Delivery** (PMBOK 8).
EVM — не только контроль затрат, но и измерение реальной ценности
относительно вложений. Опирается на PMI Practice Standard for EVM.

## Справочник формул EVM

| Показатель | Формула | CPI/SPI норма | Интерпретация при отклонении |
|---|---|---|---|
| CV | EV − AC | ≥ 0 | < 0: тратим больше, чем осваиваем |
| SV | EV − PV | ≥ 0 | < 0: осваиваем медленнее плана |
| CPI | EV / AC | ≥ 1.0 | 0.9–1.0: внимание; < 0.9: критично |
| SPI | EV / PV | ≥ 1.0 | 0.85–1.0: внимание; < 0.85: критично |
| EAC | BAC / CPI | = BAC | Прогноз итоговой стоимости |
| ETC | EAC − AC | — | Сколько ещё нужно потратить |
| TCPI | (BAC−EV)/(BAC−AC) | ≤ 1.1 | > 1.1: нереалистично уложиться в бюджет |
| VAC | BAC − EAC | ≥ 0 | < 0: ожидаемый перерасход |

## Инструкции для агента

### Шаг 1 — Получи данные EVM
Вызови `calculate_evm` и `get_budget_vs_actual`. Зафиксируй дату отчётного периода.

### Шаг 2 — Интерпретируй, не просто называй
**Плохо:** "CPI = 0.86"
**Хорошо:** "CPI = 0.86: на каждый рубль освоённого объёма проект тратит 1.16 руб.
При сохранении тенденции EAC = BAC / 0.86 = X, превышение бюджета составит Y%."

Пороги эскалации:
- CPI < 0.9 → критический перерасход, отчёт спонсору.
- SPI < 0.85 → серьёзное отставание, анализ критического пути.
- TCPI > 1.1 → нереалистичный бюджет, рекомендуй пересмотр EAC или scope.

### Шаг 3 — Проверь тренды
`analyze_kpi_trends`. Один плохой период — аномалия. Три подряд — системная проблема.

### Шаг 4 — Свяжи с бизнес-ценностью
`get_okr_alignment`. Высокий OKR-вклад при перерасходе = аргумент для
дополнительного финансирования, а не для закрытия проекта. Включай этот контекст.

### Шаг 5 — Рассчитай три сценария EAC
- Оптимистичный: CPI улучшится до 1.0 → EAC = AC + (BAC − EV)
- Реалистичный: текущий CPI сохранится → EAC = BAC / CPI
- Пессимистичный: CPI ухудшится на 10% → EAC = BAC / (CPI × 0.9)

### Шаг 6 — Выбери intent для виджета
- "Как развивался разрыв между планом и фактом?" → `evm_curves`
- "Улучшается или ухудшается эффективность?" → `cpi_spi_trend`
- "Сколько будет стоить проект в итоге?" → `eac_forecast_scenarios`
