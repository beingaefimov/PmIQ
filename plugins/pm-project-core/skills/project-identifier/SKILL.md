# Project Identifier

## Инструкция
Эти знания используем для разрешения нечётких упоминаний проектов в запросах пользователей.

### Когда вызывать ПЕРВЫМ:
- Если пользователь упоминает название проекта (даже приблизительно)
- Если пользователь спрашивает "какой проект..." или "найди проект..."
- Если в запросе есть контекст, связанный с конкретным проектом

### Как использовать результат:
После получения `project_id` из этого инструмента, используйте его как фильтр в последующих вызовах других инструментов.

### Примеры:
- Пользователь: "Покажи риски в Строительстве Фаза 1"
  → Вызовите `identify_project(query="Строительство Фаза 1")` → получите `project_id="Construction_Phase1"`
  → Затем вызовите, например `get_risk_register(project_id="Construction_Phase1")`

- Пользователь: "Какие есть перегрузки в проекте по внедрению?"
  → Вызовите `identify_project(query="проект по внедрению")` → получите список кандидатов
  → Если один кандидат — используйте его, если несколько — уточните у пользователя

- Пользователь: "Покажи статус вех в мобильном приложении"
  → Вызовите `identify_project(query="мобильное приложение")` → получите `project_id="Mobile_App_Dev"`
  → Затем вызовите `get_milestone_status(project_id="Mobile_App_Dev")`

## Required Tools
- `identify_project`
