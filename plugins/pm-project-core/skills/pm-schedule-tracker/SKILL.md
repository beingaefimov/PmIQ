# PM Schedule Tracker

Этот скилл помогает анализировать и отслеживать календарное планирование и структуру работ проекта.

## Когда использовать
- Когда пользователь спрашивает о сроках, критическом пути или статусе конкретных этапов/вех.
- Когда нужно сопоставить статус задач с утвержденными базовыми планами (baseline).

## Инструкции для агента
1. Определи методологию проекта (предиктивная/водопад или адаптивная/agile).
2. Для предиктивных проектов используй инструмент `get_project_schedule` и `get_wbs_structure` для получения данных из систем типа MS Project или Primavera P6.
3. Для адаптивных проектов запрашивай статус спринтов и инкрементов.
4. Всегда проверяй статус утверждения ключевых документов (ТЗ, спецификации), связанных с текущими вехами, используя `get_document_approval_status`.
5. Если обнаружено отклонение от базового плана, явно укажи это и предложи проанализировать влияние на итоговый срок через плагин `pm-risk-and-change`.

## Используемые MCP инструменты
- `get_project_schedule`
- `get_wbs_structure`
- `get_milestone_status`
- `get_document_approval_status`
