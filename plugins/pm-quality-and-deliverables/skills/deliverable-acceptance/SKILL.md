# Deliverable Acceptance

Контроль формальной сдачи результатов. Принцип: работа не завершена,
пока не подписан акт приемки заказчиком.
Задержка приемки → риск срыва сроков.

## Методологический контекст

### Процедура приемки (типовая)
1. **Подготовка** → проверка соответствия критериям приемки.
2. **Внутренний контроль** → проверка QA/техническим комитетом.
3. **Предъявление заказчику** → направление на согласование.
4. **Замечания (если есть)** → устранение, повторная сдача.
5. **Подписание акта** → формальное завершение.

### Статусы приемки
| Статус | Значение | Действие |
|---|---|---|
| Accepted | Акт подписан | Закрыть задачу, зафиксировать milestone |
| Pending Acceptance | Ожидает решения | Уточнить сроки у приемщика |
| In Review | На проверке | Готовить ответы на вопросы |
| Rejected | Отклонено | Анализ замечаний, план устранения |

## Инструкции для агента

### Шаг 1 — Получи статус приемки
Вызови `get_deliverable_status` для проекта.

### Шаг 2 — Выяви проблемные зоны
- `status = Pending Acceptance` > 5 дней → эскалация.
- `status = Rejected` → запросить причины и замечания.
- Длительное `In Review` → возможно, приемщик перегружен.

### Шаг 3 — Свяжи с расписанием
Проверь `get_milestone_status` — блокирует ли задержка приемки следующие этапы?

### Шаг 4 — Предложи действия
- Pending → назначить встречу с приемщиком, уточнить blockers.
- Rejected → создать задачи на устранение в `get_defect_log`.
- Длительная приемка → эскалировать спонсору.

### Шаг 5 — Выбери виджет
- "Какие результаты на приемке?" → `pending_acceptance_details`
- "Общая статистика по приемке?" → `deliverable_acceptance_status`

## Required Tools
- `get_deliverable_status`
- `get_milestone_status`
