# Порядок модернизации скиллов в PM IQ

## Архитектурная карта: что связано со скиллом

```
plugins/<plugin>/skills/<skill>/
├── SKILL.md                    ← Фронтматтер (name, description, available_widgets)
├── <tool_name>.csv             ← Мок-данные для инструмента
└── (другие CSV для других инструментов)

plugins/<plugin>/.mcp.json      ← Декларация инструментов плагина
servers/unified_mock_server.py  ← Генераторы виджетов (WIDGET_GENERATORS)
samples/pm_iq_agent.py          ← НЕ ТРОГАТЬ (всё декларативно)
```

**Ключевой принцип:** Код агента (`pm_iq_agent.py`) **никогда не меняется** при добавлении новых скиллов, инструментов или виджетов. Всё управляется через данные.

---

## Что может меняться при модернизации скилла

| Что меняется | Где отражается |
|--------------|----------------|
| Новые инструменты | `.mcp.json` плагина + новые CSV-файлы |
| Новые типы виджетов | `WIDGET_GENERATORS` в `unified_mock_server.py` |
| Новые виджеты для существующих инструментов | `available_widgets` в `SKILL.md` |
| Изменение описания/триггеров скилла | `description` в `SKILL.md` |
| Изменение структуры данных | CSV-файлы + генераторы виджетов |
| Новые параметры инструментов | `inputSchema` в `.mcp.json` |

## Порядок модернизации (чек-лист)

### Шаг 1. Анализ различий между старым и новым скиллом

Я сравниваю два `SKILL.md` и определяю, что изменилось:

```
СТАРЫЙ vs НОВЫЙ:
□ Изменилось описание (description)?
□ Добавились/удалились триггеры?
□ Изменился список available_widgets?
□ Появились новые типы виджетов (не BarChart/PieChart/ScatterChart/ActionCard)?
□ Изменились инструкции (Instructions)?
```

### Шаг 2. Определение затронутых компонентов

На основе различий определяю, какие файлы нужно изменить:

| Изменение в SKILL.md | Что нужно изменить |
|----------------------|---------------------|
| Новые `available_widgets` с **известным** типом (BarChart, PieChart, ScatterChart, ActionCard) | Только `SKILL.md` — больше ничего |
| Новые `available_widgets` с **новым** типом (GanttChart, Heatmap, и т.д.) | `SKILL.md` + новый генератор в `unified_mock_server.py` + запись в `WIDGET_GENERATORS` |
| Новые инструменты (новые CSV-файлы) | `SKILL.md` + `.mcp.json` + новый CSV-файл |
| Изменение `inputSchema` инструмента | `.mcp.json` плагина |
| Изменение `description` | Только `SKILL.md` |
| Изменение `Instructions` | Только `SKILL.md` |

### Шаг 3. Формирование плана изменений

Я выдаю отчёт в формате:

```
ПЛАН МОДЕРНИЗАЦИИ СКИЛЛА: <skill_name>

1. ФАЙЛЫ, КОТОРЫЕ НУЖНО ИЗМЕНИТЬ:
   - [ ] plugins/<plugin>/skills/<skill>/SKILL.md
   - [ ] plugins/<plugin>/.mcp.json
   - [ ] servers/unified_mock_server.py
   - [ ] plugins/<plugin>/skills/<skill>/<new_tool>.csv (новый файл)

2. НОВЫЕ ГЕНЕРАТОРЫ ВИДЖЕТОВ:
   - generate_echarts_<new_type>() — добавить в unified_mock_server.py
   - Добавить запись в WIDGET_GENERATORS

3. НОВЫЕ ИНСТРУМЕНТЫ:
   - <tool_name>: <описание> → добавить в .mcp.json
   - <tool_name>.csv: создать мок-данные

4. ФАЙЛЫ, КОТОРЫЕ НЕ ТРОГАЕМ:
   - samples/pm_iq_agent.py (всё декларативно)
```

### Шаг 4. Реализация изменений

В этом порядке:
1. Обновить `SKILL.md`
2. Обновить `.mcp.json` (если добавились инструменты)
3. Создать/обновить CSV-файлы
4. Добавить новые генераторы в `unified_mock_server.py` (если нужны новые типы виджетов)
5. Протестировать через `test_tool.py`

## Шаблон задания для меня

Когда вы даёте задание на модернизацию, используйте этот формат:

```
ЗАДАЧА: Модернизировать скилл <skill_name>

СТАРЫЙ SKILL.md:
[вставить содержимое]

НОВЫЙ SKILL.md:
[вставить содержимое]

ДОПОЛНИТЕЛЬНО (если нужно):
- Описание нового типа виджета: <что должна показывать диаграмма>
- Структура данных в CSV: <колонки и их смысл>
- Примеры мок-данных: <2-3 строки>
```

---

## Что я сделаю, получив такое задание

1. **Сравню** старый и новый `SKILL.md` построчно
2. **Определю** тип изменений (по таблице выше)
3. **Выдам** план модернизации с чек-листом файлов
4. **Попрошу подтверждения** перед изменением кода
5. **Реализую** изменения в указанном порядке
6. **Предложу** тестовый запрос для проверки

## Примеры типичных сценариев

### Сценарий A: Добавили новый виджет известного типа
**Изменения:** Только `SKILL.md` (добавить запись в `available_widgets`)

### Сценарий B: Добавили новый тип виджета (например, GanttChart)
**Изменения:** 
- `SKILL.md` (добавить в `available_widgets`)
- `unified_mock_server.py` (добавить `generate_echarts_ganttchart` и запись в `WIDGET_GENERATORS`)

### Сценарий C: Добавили новый инструмент в скилл
**Изменения:**
- `SKILL.md` (если нужно упомянуть в instructions)
- `.mcp.json` (добавить инструмент)
- Новый CSV-файл

### Сценарий D: Изменили структуру данных (колонки CSV)
**Изменения:**
- CSV-файл (новые колонки)
- Генератор виджета (если использует эти колонки)
- Возможно `SKILL.md` (если изменилась семантика)

---

## Важные ограничения

1. **Никогда не трогаем** `samples/pm_iq_agent.py` — всё должно быть декларативно
2. **Никогда не хардкодим** названия инструментов или плагинов в коде
3. **Все описания** виджетов — только в `available_widgets` в `SKILL.md`
4. **Все генераторы** — только в `unified_mock_server.py` через `WIDGET_GENERATORS`
5. **Все инструменты** — только в `.mcp.json` плагина

Этот порядок гарантирует, что система остаётся декларативной и легко расширяемой.
