# PM IQ Standalone Agent Sample

Этот пример демонстрирует, как использовать слой PM IQ (Universal Edition) с локальной языковой моделью (LLM) без зависимости от GitHub Copilot CLI.

## Архитектура

1. **LLM Server**: `llama.cpp` запускает локальную модель (например, Llama-3-8B-Instruct) и предоставляет OpenAI-совместимый API.
2. **Agent (Python)**: Реализует паттерн ReAct (Reasoning and Acting). Он решает, какой инструмент вызвать, парсит ответ и формирует финальный вывод.
3. **MCP Layer**: Python-скрипт подключается к MCP-серверу PM IQ через `stdio` и вызывает реальные инструменты (Jira, SAP, Primavera и т.д.).

## Предварительные требования

1. **Python 3.10+**
2. **Node.js и npm** (для запуска MCP-сервера через `npx`, как указано в `server.json`)
3. **Запущенный сервер llama.cpp**

### Шаг 1: Запуск llama.cpp server

Скачайте или скомпилируйте `llama.cpp`. Запустите сервер с инструктивной моделью (рекомендуется `Llama-3-8B-Instruct` или `Mistral-7B-Instruct`):

```bash
# Пример для Linux/macOS
./llama-server -m models/llama-3-8b-instruct.gguf -c 4096 --port 8080
```

*Убедитесь, что сервер отвечает на `http://localhost:8080/v1/models`.*

### Шаг 2: Установка зависимостей Python

```bash
cd samples
pip install -r requirements.txt
```

### Шаг 3: Запуск агента

```bash
python pm_iq_agent.py
```

## Как это работает (Поток ReAct)

1. Пользователь задает вопрос: *"Каков статус критического пути?"*
2. Агент формирует промпт со списком всех доступных инструментов из PM IQ.
3. LLM генерирует:
   ```text
   Thought: Мне нужно получить расписание проекта.
   Action: get_project_schedule
   Action Input: {"project_name": "Миграция ERP"}
   ```
4. Python-скрипт перехватывает это, вызывает реальный MCP-инструмент `get_project_schedule`.
5. MCP возвращает JSON с данными из Primavera/Jira.
6. Python добавляет это как `Observation` в историю чата.
7. LLM анализирует данные и выдает `Final Answer` на естественном языке.

## Настройка под вашу среду

В файле `pm_iq_agent.py` измените:
- `LLM_BASE_URL`, если ваш `llama.cpp` запущен на другом порту или хосте.
- `MCP_COMMAND` и `MCP_ARGS`, если вы запускаете MCP-сервер локально из исходного кода Python, а не через `npx`.
- Словарь `env` в `StdioServerParameters`, добавив реальные API-ключи ваших корпоративных систем.

## Важные замечания

### Почему ReAct, а не Function Calling?

Локальные модели (особенно 7B-8B параметры) часто плохо справляются со строгим JSON-форматом OpenAI Function Calling. Текстовый формат ReAct:
- Проще для парсинга (обычные regex)
- Более устойчив к галлюцинациям в формате
- Позволяет модели "рассуждать вслух" (Thought), что улучшает качество решений

### Обработка ошибок

Агент корректно обрабатывает:
- Неверный JSON в `Action Input` → возвращает ошибку в LLM для повторной попытки
- Несуществующие инструменты → информирует LLM о доступных инструментах
- Превышение итераций → возвращает вежливый отказ

### Масштабирование

Для продакшен-использования рассмотрите:
- Добавление кэширования ответов MCP-инструментов
- Реализацию retry-логики для нестабильных API
- Использование более мощных моделей (70B+) для сложных рассуждений
- Добавление векторной базы знаний для RAG по корпоративным документам PMBOK
